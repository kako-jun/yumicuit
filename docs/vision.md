# yumicuit 最終ビジョン

## 人類初の挑戦

**夢を「持ち帰る」ことを実現する。**

夢は他人に伝えたいものだが、写真に撮って持ち帰れない。
これは人類永遠のもどかしいテーマである。

脳に電極をつける研究では見ているものの画像化に成功しているが、
yumicuitはそれとは違う**現実的なアプローチ**を取る。

---

## 核心的なインサイト

### 夢の元ネタは現実である

夢の景色は、すべて**現実の形式の融合**である。

- あのビルは、渋谷のビルと大阪のビルが混ざっている
- あの空の色は、夕焼けと曇り空の中間だった
- あの道は、実家の近くと旅行先の道が合成されている

脳は見たことがないものを完全に創造することはできない。
夢は**現実の断片の再構成**である。

### 記憶の定着プロセスを活用する

yumicuitで夢を記録し、昼に何度も思い出して確定することで：

1. ユーザーは夢を何度も想起する
2. 記憶が定着する
3. 夢の景色の「確度」が上がる
4. 「どんなビルだったか」「どんな空だったか」が明確になる

---

## 夢の景色を再構成する

### プロセス

```
[Step 1] 断片記録（朝）
    ↓
[Step 2] ストーリー確定（昼）
    ↓
[Step 3] 景色の参照収集
    │
    ├─ ローカル写真から選択
    │   「このビルはこの写真に近い」
    │
    ├─ ネット画像検索
    │   「空の色はこんな感じ」
    │
    └─ 複数の参照を組み合わせ
        「このビルとこの空と、この道を合成」
    ↓
[Step 4] AI画像合成
    ↓
[Step 5] 確認・調整
    「もう少し暗く」「ビルをもっと高く」
    ↓
[Step 6] 完成
    「これが私の見た夢の景色です」
```

### 参照画像の扱い

- **ローカル写真**：ユーザーのカメラロールから選択
- **ネット画像**：検索して選択
- **保存しない**：入力として使うだけ。著作権の問題を回避
- **生成結果のみ保存**：AI合成された「夢の景色」だけを残す

### 入力例

```
ビル：[写真A] + [写真B] の融合
    「Aの形だけどBの色」

空：[写真C]
    「この夕焼けに近いけど、もっと紫がかっていた」

道：[写真D] + [写真E]
    「Dの幅でEの舗装」

全体の雰囲気：[写真F]
    「この映画のワンシーンみたいな感じ」
```

### AI合成

ローカルの画像生成AI、またはオンラインサービスで：

1. 参照画像を入力
2. 「これらを融合して」というプロンプト
3. 生成 → 確認 → 調整 → 再生成
4. 納得したら確定

## 技術的なアプローチ

### Phase 1-3（現在の計画）

テキストベース：断片→ストーリー

### Phase 4以降（将来）

画像生成の統合：

```
テキスト断片
    +
参照画像群
    ↓
AI画像合成
    ↓
夢の景色
```

### 候補技術

| 技術 | 特徴 |
|------|------|
| Stable Diffusion (ローカル) | プライバシー◎、GPU必要 |
| ControlNet | 構図制御に強い |
| IP-Adapter | 参照画像のスタイル転写 |
| DALL-E / Midjourney (オンライン) | 高品質だが従量課金 |

**ローカル完結を目指すが、画像生成は負荷が高い。**
スマホでの実現は現時点では困難。PC版での対応が現実的。

## 制約と割り切り

### 完全な再現は目指さない

- 夢は曖昧なもの
- 「近い」「だいたいこんな感じ」で十分
- 100%の再現より、**伝えられること**が重要

### プライバシーの維持

- 参照画像は保存しない
- 生成結果は本人のもの
- 共有は本人の選択

### 著作権への配慮

- 参照画像はAI入力に使うだけ
- 生成結果は新規の創作物
- ネット画像は保存しない（一時的な参照のみ）

### 個人ツールでは定着しない

創作家やアイデアをメモする人間にしか使ってもらえない。

### 「他人に見せられる」が本質

夢の景色を**SNSで世界中に共有できる**。

- 現実ではお金持ちでない人も
- 学生も
- 日本人以外も
- 誰でも、自分だけの「夢の景色」を持っている

**新しい当たり前の趣味**として、ジャンルを作れるレベルのヒットを狙う。

### 「修行」という文化

Dream Cameraで夢の景色を生成するには、入力が必要。
その入力とは、**毎日地道に夢を半覚醒でメモること**。

これを**「修行」**と呼ぶ。

```
修行（毎朝の断片記録）
    ↓
記憶の定着
    ↓
夢の景色の確度が上がる
    ↓
Dream Cameraで再構成
    ↓
SNSで共有
    ↓
「これが私の夢です」
```

修行を積むほど、夢を鮮明に思い出せるようになる。
修行を積むほど、Dream Cameraの精度が上がる。
修行を積むほど、共有できる夢が増える。

### 独特の文化

- **#DreamCamera** - 夢の景色を共有するハッシュタグ
- **#夢修行** - 毎朝の記録を続ける習慣
- **夢師（ゆめし）** - 修行を極めた人
- **夢景（むけい）** - 生成された夢の景色

これは単なるアプリではない。
**新しい趣味、新しい文化、新しい自己表現の形**。
