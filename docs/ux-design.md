# yumicuit UXデザイン

## 3つのモード

yumicuitは3つの完全に分離されたモードを持つ。

```
起床直後 ──→ まどろみ記録モード（黒紙）
    │
    │ （時間経過）
    ↓
昼間 ──────→ ふりかえり生成モード（AI作文）
    │
    │ （時間経過）
    ↓
就寝前 ────→ 就寝前プライミングモード（夢の種）
```

---

## 1. まどろみ記録モード（起床直後）

### 目的

覚醒度を上げない。断片だけ残して、忘れて二度寝してOK。

### フロー

```
起動
  ↓
即、入力画面（黒紙 + 白文字）
  ↓
断片入力
  ↓
数秒無入力
  ↓
自動保存 → 画面暗転
```

**「楽しい？」は聞かない。**
楽しい夢でなければそもそも起動しない。余計な操作を増やさない。

### 入力の仕方

- **単語の断片だけ**でいい
- 改行1回 = 次のトークン（文章は後でAIが作る）
- 例：
  ```
  港
  緑の鳥
  雷
  笑った
  追いかけっこ
  ```

### 視覚デザイン

- **背景**：完全な黒（#000000）
- **文字**：抑えめな白（#AAAAAA〜#BBBBBB程度）
  - 純粋な白（#FFFFFF）は眩しいので使わない
  - 暗い部屋でも目に優しい
- **行間**：広め（読みやすく）
- **表示要素**：カーソルのみ
- 枠なし、ボタンなし、アイコンなし、ロゴなし、日付なし

**輝度について**：
- ユーザーの端末設定は触らない
- アプリ側のテーマで対応（黒背景 + 抑えめ白文字）
- これなら輝度が高くても眩しくない

### インタラクション

- **保存は無音オート**：数秒無入力で自動保存
- **保存完了**：触覚のみ（振動1回）、音なし、視覚フィードバック最小
- **画面暗転**：保存後、アプリが勝手に閉じる設計

### 音声入力（オプション）

- 端末ネイティブの音声キーボードを使用
- Web Speech API はブラウザ差が大きいため必須機能にしない
- 一言だけ喋って断片を残せる

### 禁止事項（絶対守る）

1. 朝の画面に履歴を見せない
2. 通知・バッジは一切使わない
3. 文章入力を強制しない（断片のみでOK）
4. ロゴやブランディング要素を見せない
5. 設定画面への導線を見せない
6. 余計な質問をしない（「楽しい？」など聞かない）

## 2. ふりかえり生成モード（昼）

### 目的

断片から意味を再構成。納得したら「夢の核」として固定。

### フロー

```
断片を選択
  ↓
AIが3案の短編を生成（150-300字）
  ↓
パラメータ調整（任意）
  ├─ 強引さスライダー（低=保守的 / 高=創造的）
  └─ 連想方向（場所/人/感情/色/音）
  ↓
合ってなければ「やり直し」
  ↓
「こんな夢だった」→ 確定
  ↓
タグ付与（場所/感情/季節など、任意）
```

### AIの「強引作文」

- 断片だけでは意味が分からない
- AIが**無理やり**ストーリーを作る
- 3案出して、ユーザーが「これだ」と選ぶ
- 合ってなければパラメータ変えてやり直し

### 生成パラメータ

#### 強引さ（force）
- 0.0-1.0
- 低い：断片に忠実、保守的
- 高い：断片から飛躍、創造的

#### 連想方向（focus）
- `place`：場所を中心に展開
- `person`：人物を中心に展開
- `emotion`：感情を中心に展開
- `color`：色彩を中心に展開
- `sound`：音を中心に展開

### 確定後

- 確定したストーリーは「夢の核」として保存
- 生成のやり直し履歴も残す（差分として）
- タグは任意（付けなくてもいい）

### 広告表示

- **このモードのみ**広告を表示（将来）
- AI生成ストーリーの下に配置
- 朝モード・就寝前モードには絶対に広告を入れない

---

## 3. 就寝前プライミングモード

### 目的

再夢の確率を上げる「記憶の種まき」。刺激は最小。

### 表示内容：「夢の種（Dream Seeds）」

```
港｜緑の鳥｜雷｜笑った｜追いかけっこ

「雷鳴の海で、緑の鳥を追いかけて笑った。」
```

- キーワード5-7個
- 核の一文（確定したストーリーから抽出）

### 視覚デザイン

- 黒背景維持
- 文字は薄めの白（#CCCCCC程度）
- スクロールなし（1画面に収める）
- 文字微小にじみ（視覚的に眠気を誘う）
- アニメーションは極小（呼吸テンポの円の拡張収縮のみ、任意）

### インタラクション

- 操作はほぼ不要
- 見るだけ
- 「あの続きが見たい」と思いながら画面オフ→睡眠

### 禁止事項

1. 広告ゼロ（睡眠導入を妨げない）
2. 長文禁止（語数制限）
3. 明るい色禁止
4. 通知禁止

---

## インタラクション原則（全モード共通）

### 3秒総量

起動→選択→断片入力→自動保存→暗転まで**3秒台を狙う**。

### 黒紙のみ

- アイコン非表示
- ロゴ非表示
- 日付非表示
- メーター類非表示
- 設定ボタン非表示（朝は）

### 無通知

- バナー禁止
- バッジ禁止
- プッシュ通知禁止
- ホームショートカットから即起動のみ

### 暗所最適

- 黒背景 + 抑えめ白文字（#AAAAAA〜#BBBBBB）
- 純粋な白（#FFFFFF）は使わない
- ユーザーの輝度設定は触らない（アプリ側テーマで対応）
- 音も光も最小

### 手触り（触覚フィードバック）

- YES/NOタップ：短い触覚
- 保存完了：短い触覚
- それ以外は触覚なし

### アニメーション

- フェード120-180ms以内
- 派手なトランジション禁止
- 動きで覚醒させない

---

## 起動導線

### スマホ（主）

- **ホーム画面ショートカット**（PWA Install）から即起動
- iOS：Siriショートカット「楽しい夢」→ URL Schemeで即入力画面
- Android：クイック設定タイル or ホームショートカット

### PC（従）

- PWAなので自動的にPC対応
- ただし主な用途は昼のふりかえりモードのみ
- 朝・就寝前はスマホ前提

---

## マイクロコピー

最小限の言葉だけ。基本的に何も表示しない。

| 場面 | コピー |
|------|--------|
| 起動直後 | （なし、即入力画面） |
| 入力中 | （なし） |
| 保存完了 | （触覚のみ、テキストなし） |
| 就寝前 | キーワード列＋核の一文のみ |

---

## 失敗しないための禁止事項（まとめ）

1. 朝の画面に履歴を見せない
2. 通知・バッジは一切使わない
3. 文章入力の強制をしない（断片のみでOK）
4. 朝と就寝前に広告を入れない
5. 設定画面を朝に見せない
6. 統計や分析を朝に見せない
7. 派手なアニメーションを使わない
8. 音を出さない
9. 明るい色を使わない
10. 余計な質問をしない（起動→即入力）
11. 昼のUIも控えめに（覚醒してもいいが統一感を保つ）
