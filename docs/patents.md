# yumicuit 技術公開

## 概要

yumicuitには3つの核心技術がある。

先行技術調査の結果、これらは既存特許と重複しない新規性のある技術であることを確認した。
しかし、特許による独占ではなく、**防衛的公開**を選択し、技術を公開する。

---

## 防衛的公開を選択した理由

### 1. 技術の社会貢献

夢を記録・再現する技術は人類共通の願望である。
特許で独占するより、広く使われる方が価値がある。

### 2. 先行者利益で十分

競争力の源泉は技術だけではない：

- **UX設計**: 半覚醒状態への配慮は真似しにくい
- **ブランド**: 「yumicuit」「Dream Camera」「修行」という文化
- **データ**: ユーザーの蓄積した夢記録は他アプリに移行しにくい
- **ノウハウ**: AIプロンプトの調整、継続的な改善

### 3. オープンソースの精神

技術を公開することで、他の開発者も改良・発展させられる。
エコシステム全体の発展に貢献する。

### 防衛的公開の効果

この技術をGitHubで公開することにより：

- **誰も特許を取れなくなる**（公知技術となるため）
- yumicuit自身も特許は取れないが、他者に独占されることもない

## 技術1: 半覚醒入力システム

### 技術名称

「睡眠覚醒移行状態における記憶情報の非誘導型収集システム」

### 課題

- 起床直後の夢の記憶は数分で消失する
- 従来の夢日記アプリは質問やUIで誘導するため、脳が覚醒し記憶が上書きされる
- 誘導なしでは必要な情報（色、空間、感情等）が記録されない

### 解決手段

1. **起動時に何も表示しない**
   - 質問、プロンプト、プレースホルダーを一切表示しない
   - 黒背景 + 抑えめ白文字のみの最小UI
   - ユーザーの脳を覚醒させない

2. **自由記述のみを許容**
   - 構造化入力を強制しない
   - 改行区切りの断片入力
   - 数秒無入力で自動保存・自動終了

3. **昼の補完による学習効果**
   - 昼にAIが「足りない情報」を質問
   - 質問を繰り返すうちに、ユーザーは朝に自然と豊かな情報を書くようになる
   - この「修行」により、誘導なしで高品質な入力が得られる

4. **入力と出力の分離**
   - ユーザー入力は不可侵として保存
   - AI生成結果は別管理
   - 昼の追記（「そういえばこうだった」）をいつでも可能に

### 効果

- 覚醒度を上げずに夢の記憶を保持
- 誘導なしで画像生成に必要な詳細情報を収集
- 継続使用により入力品質が自然と向上

## 技術2: 夢景画像再構成システム（Dream Camera）

### 技術名称

「断片的記憶情報と参照画像群からの主観的視覚体験再構成システム」

### 課題

- 夢の景色は写真に撮れない
- 言葉だけでは他人に伝えられない
- 脳に電極をつける方法は非現実的

### 解決手段

1. **夢の元ネタは現実であるという洞察**
   - 夢の景色は現実の断片の再構成
   - 「このビルとあの空の組み合わせ」で表現可能

2. **構造化された夢情報の活用**
   - 技術1で収集した詳細情報を使用
   - 時間帯、天気、光源、色調、空間の広さ、視点など
   - これらを画像生成プロンプトに変換

3. **参照画像の組み合わせ指定**
   - ユーザーがローカル写真やネット画像から参照を選択
   - 「このビルの形 + この空の色 + この道の雰囲気」
   - 複数参照の融合指示をAIに伝達

4. **プロンプト代筆システム**
   - ユーザーの日本語指示を英語の最適プロンプトに変換
   - 参照画像の添付を代行
   - 各種画像生成AI（ローカル/オンライン）に対応

5. **反復的な確認・調整**
   - 生成結果をユーザーが確認
   - 「もう少し暗く」「ビルをもっと高く」等の調整
   - 納得するまで再生成

### 効果

- 夢の景色を視覚的に再現
- 他人に「これが私の夢です」と見せられる
- 人類初の「夢を持ち帰る」体験を実現

## 技術3: 夢誘導プライミングシステム

### 技術名称

「入眠時記憶刺激による目標夢体験誘導システム」

### 課題

- 見たい夢を見る方法が確立されていない
- 明晰夢の訓練は困難で成功率が低い
- 夢の続きを見たいのに見られない

### 解決手段

1. **夢の種（Dream Seeds）の生成**
   - 確定した夢から5-7個のキーワードを抽出
   - 核となる一文を生成（詩的、印象的）
   - これを「夢の種」として保存

2. **就寝前プライミング表示**
   - 黒背景に抑えめ白文字で夢の種を表示
   - スクロールなし、1画面に収める
   - 視覚刺激を最小限に抑えつつ記憶を刺激

3. **寝落ちするまで見せ続ける**
   - ユーザーが画面を見ながらうとうとする状態を想定
   - 「あの夢の続きが見たい」という思考を維持
   - 入眠直前まで夢の情報を意識させる

4. **Dream Cameraとの連携**
   - 再現した夢の景色画像を就寝前に表示
   - 視覚的なプライミングで再夢確率を向上
   - キーワード + 一文 + 画像の組み合わせ

5. **再夢の記録とフィードバック**
   - 翌朝、狙った夢が見れたか記録
   - 成功パターンを学習
   - プライミング方法を最適化

### 効果

- 狙い通りの夢を見る確率を向上
- 夢の続きを見る体験を提供
- 再夢の成功体験により継続利用を促進

---

## 3つの技術の連携

```
[技術1] 半覚醒入力
    ↓ 高品質な夢情報を収集
[技術2] Dream Camera
    ↓ 夢の景色を視覚化
[技術3] プライミング
    ↓ 視覚化した景色で再夢を誘導
[技術1] 半覚醒入力
    ↓ 再夢の内容を記録
    ... (ループ)
```

この3つが連携することで、
「夢を記録 → 再現 → 再び見る」というサイクルが完成する。

## 先行技術調査

詳細は [patents/prior-art-research.md](../patents/prior-art-research.md) を参照。

主な調査結果：

| 先行技術 | 概要 | yumicuitとの差別化 |
|----------|------|-------------------|
| US20040192996A1 | 夢記録デバイス（音声） | テキストUI、学習効果なし |
| MIT Dormio | 入眠時TDI | ハードウェア依存、睡眠中刺激あり |
| US20240307652A1 | 夢誘導システム | 睡眠中刺激あり、夢の種概念なし |
| SleepVision | AI夢画像生成 | 参照画像融合なし、メタデータなし |

## 競合への防御

### 模倣されやすい部分

- 黒背景UI（デザインの模倣は容易）
- 自動保存機能（一般的な技術）

### 模倣されにくい部分

- 「昼の補完 → 朝の入力品質向上」の学習ループ
- 構造化メタデータ + 参照画像 → 夢画像生成のパイプライン
- 夢の種 + 夢画像によるプライミング効果
- 「修行」「夢師」等の文化・ブランドの確立
- 先行者利益（ユーザーの蓄積データ）
- 継続的な改善によるノウハウ蓄積
